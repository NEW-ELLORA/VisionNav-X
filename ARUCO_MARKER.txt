==================================================
WHAT IS AN ARUCO MARKER (TECHNICAL DEFINITION)
==================================================

An ArUco marker is a binary square fiducial marker used in computer vision for robust detection and 6-DoF pose estimation.

It is implemented in OpenCV as part of the aruco module.

Each marker consists of:
- A black border (for segmentation)
- An inner binary matrix (n × n bits) encoding a unique ID
- A predefined dictionary with error correction capability


==================================================
MARKER REPRESENTATION
==================================================

A marker is defined as:

- Grid size: e.g., 4×4, 5×5, 6×6
- Each cell: binary (0 = black, 1 = white)
- Encoded ID: integer mapped from binary pattern

Example:
- 6x6 marker → 36 bits → unique ID


==================================================
DETECTION PIPELINE (ALGORITHM)
==================================================

1. Image Preprocessing
- Convert RGB → grayscale
- Apply adaptive thresholding to obtain binary image

2. Contour Detection
- Detect contours using edge detection
- Filter contours based on:
  • Quadrilateral shape
  • Convexity
  • Area threshold

3. Candidate Filtering
- Approximate contours → polygons
- Select 4-sided polygons (potential markers)
- Order corners (clockwise)

4. Perspective Transformation
- Apply homography to warp candidate to a canonical square

5. Bit Extraction
- Divide warped image into n × n grid
- Compute mean pixel intensity per cell
- Threshold → binary pattern

6. Dictionary Matching
- Compare extracted binary matrix with marker dictionary
- Use Hamming distance for matching
- Apply error correction

7. Marker Identification
- Assign ID based on closest match in dictionary
- Reject if error exceeds threshold


==================================================
POSE ESTIMATION (CORE MATHEMATICS)
==================================================

Once corners are detected:

We solve the Perspective-n-Point (PnP) problem.

Given:
- 3D coordinates of marker corners (known)
- 2D image projections (detected)

We estimate:
- Rotation vector R
- Translation vector t


--------------------------------------------------
CAMERA PROJECTION MODEL
--------------------------------------------------

x = K [ R | t ] X

Where:
- K → camera intrinsic matrix
- R → rotation matrix
- t → translation vector
- X → 3D point
- x → 2D image point

Output:
- 6-DoF pose:
  • Position → (x, y, z)
  • Orientation → (roll, pitch, yaw)


==================================================
KEY ALGORITHMS USED
==================================================

- Adaptive Thresholding
- Contour Detection (Suzuki-Abe)
- Polygon Approximation (Douglas-Peucker)
- Homography Estimation
- Hamming Distance Matching
- PnP (e.g., EPnP, Levenberg-Marquardt)


==================================================
DICTIONARY & ERROR CORRECTION
==================================================

Each dictionary:
- Contains a set of markers with maximum inter-marker distance
- Ensures robustness using minimum Hamming distance

Example:
- DICT_4X4_50 → 50 markers, 4×4 grid
- Can tolerate bit errors during detection


==================================================
ADVANTAGES (TECHNICAL)
==================================================

- Low computational cost
- Rotation invariant detection
- Robust to partial occlusion
- Real-time performance (30–60 FPS)
- Accurate pose estimation with calibrated camera


==================================================
LIMITATIONS
==================================================

- Requires camera calibration for accurate pose
- Performance drops in:
  • Motion blur
  • Low lighting
  • Extreme perspective distortion


==================================================
IN ONE TECHNICAL LINE
==================================================

ArUco markers are binary fiducial systems enabling robust marker detection and real-time 6-DoF pose estimation via contour analysis, homography, and PnP-based geometric reconstruction.
